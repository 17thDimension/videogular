videogular
videogular in module com.2fdevs.videogular

Description

Main directive that must wrap a <video> tag and all plugins.

<video> tag usually will be above plugin tags, that's because plugins should be in a layer over the <video>.

You can customize videogular with these attributes:

Usage

as element:
<videogular
       vg-width="{number or string}"
       vg-height="{number or string}"
       vg-theme="{string}"
       auto-play="{boolean or string}"
       stretch="{string}"
       is-responsive="{boolean or string}"
       vg-complete="{function}"
       vg-update-volume="{function}"
       vg-update-time="{function}"
       vg-update-size="{function}"
       vg-update-state="{function}"
       vg-player-ready="{function}"
       vg-change-source="{function}">
</videogular>

Parameters

ParamTypeDetails
vgWidthnumber or string

This directive sets width for the entire player. Passing a number will set the width normally. Passing a string will create a binding with a scope variable in case it exists.

If vgWidth or vgHeight are not declared, or vgResponsive is "true", player will enter in a responsive mode and width will be 100% and height will be calculated through video metadata to preserve aspect ratio.

vgHeightnumber or string

This directive sets height for the entire player. Passing a number will set the height normally. Passing a string will create a binding with a scope variable in case it exists.

If vgWidth or vgHeight are not declared, or vgResponsive is "true", player will enter in a responsive mode and width will be 100% and height will be calculated through video metadata to preserve aspect ratio.

vgThemestring

String with a scope name variable. This directive will inject a CSS link in the header of your page. This parameter is required.

autoPlayboolean or string

vgAutoplay Boolean value or a String with a scope name variable to auto start playing video when it is initialized.

This parameter is disabled in mobile devices because user must click on content to prevent consuming mobile data plans.

(default: false)

stretchstring

vgStretch String value representing a stretch mode. This value controls how image will scale inside its container. Stretch modes available are "none", "fit" or "fill".

  • "none": Will set the image in its original size.
  • "fit": Will try to show always all the image leaving black bars above and below.
  • "fill": Will try to cover all video player area to never show black bars above and below.

Content will always appear centered.

(default: none)

isResponsiveboolean or string

vgResponsive Boolean value or a String with a scope name variable to auto start playing video when it is initialized.

(default: false)

vgCompletefunction

Function name in controller's scope to call when video have been completed.

vgUpdateVolumefunction

Function name in controller's scope to call when volume changes. Receives a param with the new volume.

vgUpdateTimefunction

Function name in controller's scope to call when video playback time is updated. Receives two params with current time and duration in milliseconds.

vgUpdateSizefunction

Function name in controller's scope to call when videogular size is updated. Receives two param with the new width and height.

vgUpdateStatefunction

Function name in controller's scope to call when video state changes. Receives a param with the new state. Possible values are "play", "stop" or "pause".

vgPlayerReadyfunction

Function name in controller's scope to call when video have been initialized. Receives a param with the videogular API.

vgChangeSourcefunction

Function name in controller's scope to change current video source. Receives a param with the new video. This is a free parameter and it could be values like "new.mp4", "320" or "sd". This will allow you to use this to change a video or video quality. This callback will not change the video, you should do that by updating your sources scope variable.
